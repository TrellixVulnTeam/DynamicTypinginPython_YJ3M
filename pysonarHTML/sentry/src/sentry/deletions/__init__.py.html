   1 """
   2 The deletions subsystem managers bulk deletes as well as cascades. It attempts
   3 to optimize around various patterns while using a standard approach to do so.
   4 
   5 For example, let's say you want to delete an organization:
   6 
   7 >>> from sentry import deletions
   8 >>> task = deletions.get(model=Organization)
   9 >>> work = True
  10 >>> while work:
  11 >>>    work = task.chunk()
  12 
  13 The system has a default task implementation to handle Organization which will
  14 efficiently cascade deletes. This behavior varies based on the input object,
  15 as the task can override the behavior for it's children.
  16 
  17 For example, when you delete a Group, it will cascade in a more traditional
  18 manner. It will batch each child (such as Event). However, when you delete a
  19 project, it won't actually cascade to the registered Group task. It will instead
  20 take a more efficient approach of batch deleting its indirect descendants, such
  21 as Event, so it can more efficiently bulk delete rows.
  22 """
