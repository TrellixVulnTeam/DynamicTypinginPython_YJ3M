  13     """Google Protocol Buffers (AKA protobuf) is a popular data
  14     serialization scheme used for communication protocols, data storage,

  16     popular language. The focus points of this scheme are brevity (data
  17     is encoded in a very size-efficient manner) and extensibility (one
  18     can add keys to the structure, while keeping it readable in previous
  19     version of software).
  20     
  21     Protobuf uses semi-self-describing encoding scheme for its
  22     messages. It means that it is possible to parse overall structure of
  23     the message (skipping over fields one can't understand), but to
  24     fully understand the message, one needs a protocol definition file
  25     (`.proto`). To be specific:
  26     
  27     * "Keys" in key-value pairs provided in the message are identified
  28       only with an integer "field tag". `.proto` file provides info on
  29       which symbolic field names these field tags map to.
  30     * "Keys" also provide something called "wire type". It's not a data
  31       type in its common sense (i.e. you can't, for example, distinguish
  32       `sint32` vs `uint32` vs some enum, or `string` from `bytes`), but
  33       it's enough information to determine how many bytes to
  34       parse. Interpretation of the value should be done according to the
  35       type specified in `.proto` file.
  36     * There's no direct information on which fields are optional /
  37       required, which fields may be repeated or constitute a map, what
  38       restrictions are placed on fields usage in a single message, what
  39       are the fields' default values, etc, etc.
