  15 """R-FCN meta-architecture definition.
  16 
  17 R-FCN: Dai, Jifeng, et al. "R-FCN: Object Detection via Region-based
  18 Fully Convolutional Networks." arXiv preprint arXiv:1605.06409 (2016).
  19 
  20 The R-FCN meta architecture is similar to Faster R-CNN and only differs in the
  21 second stage. Hence this class inherits FasterRCNNMetaArch and overrides only
  22 the `_predict_second_stage` method.
  23 
  24 Similar to Faster R-CNN we allow for two modes: number_of_stages=1 and
  25 number_of_stages=2.  In the former setting, all of the user facing methods
  26 (e.g., predict, postprocess, loss) can be used as if the model consisted
  27 only of the RPN, returning class agnostic proposals (these can be thought of as
  28 approximate detections with no associated class information).  In the latter
  29 setting, proposals are computed, then passed through a second stage
  30 "box classifier" to yield (multi-class) detections.
  31 
  32 Implementations of R-FCN models must define a new FasterRCNNFeatureExtractor and
  33 override three methods: `preprocess`, `_extract_proposal_features` (the first
  34 stage of the model), and `_extract_box_classifier_features` (the second stage of
  35 the model). Optionally, the `restore_fn` method can be overridden.  See tests
  36 for an example.
  37 
  38 See notes in the documentation of Faster R-CNN meta-architecture as they all
  39 apply here.
  40 """

  89 
  90     Args:
  91       is_training: A boolean indicating whether the training version of the
  92         computation graph should be constructed.
  93       num_classes: Number of classes.  Note that num_classes *does not*
  94         include the background category, so if groundtruth labels take values
  95         in {0, 1, .., K-1}, num_classes=K (and not K+1, even though the
  96         assigned classification targets can range from {0,... K}).
  97       image_resizer_fn: A callable for image resizing.  This callable always
  98         takes a rank-3 image tensor (corresponding to a single image) and
  99         returns a rank-3 image tensor, possibly with new spatial dimensions.
 100         See builders/image_resizer_builder.py.
 101       feature_extractor: A FasterRCNNFeatureExtractor object.
 102       number_of_stages:  Valid values are {1, 2}. If 1 will only construct the
 103         Region Proposal Network (RPN) part of the model.
 104       first_stage_anchor_generator: An anchor_generator.AnchorGenerator object
 105         (note that currently we only support
 106         grid_anchor_generator.GridAnchorGenerator objects)
 107       first_stage_target_assigner: Target assigner to use for first stage of
 108         R-FCN (RPN).
 109       first_stage_atrous_rate: A single integer indicating the atrous rate for
 110         the single convolution op which is applied to the `rpn_features_to_crop`
 111         tensor to obtain a tensor to be used for box prediction. Some feature
 112         extractors optionally allow for producing feature maps computed at
 113         denser resolutions.  The atrous rate is used to compensate for the
 114         denser feature maps by using an effectively larger receptive field.
 115         (This should typically be set to 1).
 116       first_stage_box_predictor_arg_scope_fn: Either a
 117         Keras layer hyperparams object or a function to construct tf-slim
 118         arg_scope for conv2d, separable_conv2d and fully_connected ops. Used
 119         for the RPN box predictor. If it is a keras hyperparams object the
 120         RPN box predictor will be a Keras model. If it is a function to
 121         construct an arg scope it will be a tf-slim box predictor.
 122       first_stage_box_predictor_kernel_size: Kernel size to use for the
 123         convolution op just prior to RPN box predictions.
 124       first_stage_box_predictor_depth: Output depth for the convolution op
 125         just prior to RPN box predictions.
 126       first_stage_minibatch_size: The "batch size" to use for computing the
 127         objectness and location loss of the region proposal network. This
 128         "batch size" refers to the number of anchors selected as contributing
 129         to the loss function for any given image within the image batch and is
 130         only called "batch_size" due to terminology from the Faster R-CNN paper.
 131       first_stage_sampler: The sampler for the boxes used to calculate the RPN
 132         loss after the first stage.
 133       first_stage_non_max_suppression_fn: batch_multiclass_non_max_suppression
 134         callable that takes `boxes`, `scores` and optional `clip_window`(with
 135         all other inputs already set) and returns a dictionary containing
 136         tensors with keys: `detection_boxes`, `detection_scores`,
 137         `detection_classes`, `num_detections`. This is used to perform non max
 138         suppression  on the boxes predicted by the Region Proposal Network
 139         (RPN).
 140         See `post_processing.batch_multiclass_non_max_suppression` for the type
 141         and shape of these tensors.
 142       first_stage_max_proposals: Maximum number of boxes to retain after
 143         performing Non-Max Suppression (NMS) on the boxes predicted by the
 144         Region Proposal Network (RPN).
 145       first_stage_localization_loss_weight: A float
 146       first_stage_objectness_loss_weight: A float
 147       crop_and_resize_fn: A differentiable resampler to use for cropping RPN
 148         proposal features.
 149       second_stage_target_assigner: Target assigner to use for second stage of
 150         R-FCN. If the model is configured with multiple prediction heads, this
 151         target assigner is used to generate targets for all heads (with the
 152         correct `unmatched_class_label`).
 153       second_stage_rfcn_box_predictor: RFCN box predictor to use for
 154         second stage.
 155       second_stage_batch_size: The batch size used for computing the
 156         classification and refined location loss of the box classifier.  This
 157         "batch size" refers to the number of proposals selected as contributing
 158         to the loss function for any given image within the image batch and is
 159         only called "batch_size" due to terminology from the Faster R-CNN paper.
 160       second_stage_sampler: The sampler for the boxes used for second stage
 161         box classifier.
 162       second_stage_non_max_suppression_fn: batch_multiclass_non_max_suppression
 163         callable that takes `boxes`, `scores`, optional `clip_window` and
 164         optional (kwarg) `mask` inputs (with all other inputs already set)
 165         and returns a dictionary containing tensors with keys:
 166         `detection_boxes`, `detection_scores`, `detection_classes`,
 167         `num_detections`, and (optionally) `detection_masks`. See
 168         `post_processing.batch_multiclass_non_max_suppression` for the type and
 169         shape of these tensors.
 170       second_stage_score_conversion_fn: Callable elementwise nonlinearity
 171         (that takes tensors as inputs and returns tensors).  This is usually
 172         used to convert logits to probabilities.
 173       second_stage_localization_loss_weight: A float
 174       second_stage_classification_loss_weight: A float
 175       second_stage_classification_loss: A string indicating which loss function
 176         to use, supports 'softmax' and 'sigmoid'.
 177       hard_example_miner:  A losses.HardExampleMiner object (can be None).
 178       parallel_iterations: (Optional) The number of iterations allowed to run
 179         in parallel for calls to tf.map_fn.
 180       add_summaries: boolean (default: True) controlling whether summary ops
 181         should be added to tensorflow graph.
 182       clip_anchors_to_image: The anchors generated are clip to the
 183         window size without filtering the nonoverlapping anchors. This generates
 184         a static number of anchors. This argument is unused.
 185       use_static_shapes: If True, uses implementation of ops with static shape
 186         guarantees.
 187       resize_masks: Indicates whether the masks presend in the groundtruth
 188         should be resized in the model with `image_resizer_fn`
 189       freeze_batchnorm: Whether to freeze batch norm parameters during
 190         training or not. When training with a small batch size (e.g. 1), it is
 191         desirable to freeze batch norm update and use pretrained batch norm
 192         params.
 193       return_raw_detections_during_predict: Whether to return raw detection
 194         boxes in the predict() method. These are decoded boxes that have not
 195         been through postprocessing (i.e. NMS). Default False.
 196       output_final_box_features: Whether to output final box features. If true,
 197         it crops the feauture map based on the final box prediction and returns
 198         in the dict as detection_features.
 199 
 200     Raises:
 201       ValueError: If `second_stage_batch_size` > `first_stage_max_proposals`
 202       ValueError: If first_stage_anchor_generator is not of type
 203         grid_anchor_generator.GridAnchorGenerator.
 204     """
 205     # TODO(rathodv): add_summaries and crop_and_resize_fn is currently
 206     # unused. Respect that directive in the future.
 207     super(RFCNMetaArch, self).__init__(
 208         is_training,
 209         num_classes,
 210         image_resizer_fn,
 211         feature_extractor,
 212         number_of_stages,
 213         first_stage_anchor_generator,
 214         first_stage_target_assigner,
 215         first_stage_atrous_rate,
 216         first_stage_box_predictor_arg_scope_fn,
 217         first_stage_box_predictor_kernel_size,
 218         first_stage_box_predictor_depth,

 236         second_stage_classification_loss_weight,
 237         second_stage_classification_loss,
 238         1.0,  # second stage mask prediction loss weight isn't used in R-FCN.
 239         hard_example_miner,
 240         parallel_iterations,
 241         add_summaries,
 242         clip_anchors_to_image,
 243         use_static_shapes,
 244         resize_masks,
 245         freeze_batchnorm=freeze_batchnorm,
 246         return_raw_detections_during_predict=(
 247             return_raw_detections_during_predict),

 252   def _predict_second_stage(self, rpn_box_encodings,
 253                             rpn_objectness_predictions_with_background,
 254                             rpn_features,
 255                             anchors,
 256                             image_shape,
 257                             true_image_shapes):
 258     """Predicts the output tensors from 2nd stage of R-FCN.
 259 
 260     Args:
 261       rpn_box_encodings: 3-D float tensor of shape
 262         [batch_size, num_valid_anchors, self._box_coder.code_size] containing
 263         predicted boxes.
 264       rpn_objectness_predictions_with_background: 3-D float tensor of shape
 265         [batch_size, num_valid_anchors, 2] containing class
 266         predictions (logits) for each of the anchors.  Note that this
 267         tensor *includes* background class predictions (at class index 0).
 268       rpn_features: A 4-D float32 tensor with shape
 269         [batch_size, height, width, depth] representing image features from the
 270         RPN.
 271       anchors: 2-D float tensor of shape
 272         [num_anchors, self._box_coder.code_size].
 273       image_shape: A 1D int32 tensors of size [4] containing the image shape.
 274       true_image_shapes: int32 tensor of shape [batch, 3] where each row is
 275         of the form [height, width, channels] indicating the shapes
 276         of true images in the resized images, as resized images can be padded
 277         with zeros.
 278 
 279     Returns:
 280       prediction_dict: a dictionary holding "raw" prediction tensors:
 281         1) refined_box_encodings: a 3-D tensor with shape
 282           [total_num_proposals, num_classes, 4] representing predicted
 283           (final) refined box encodings, where
 284           total_num_proposals=batch_size*self._max_num_proposals
 285         2) class_predictions_with_background: a 2-D tensor with shape
 286           [total_num_proposals, num_classes + 1] containing class
 287           predictions (logits) for each of the anchors, where
 288           total_num_proposals=batch_size*self._max_num_proposals.
 289           Note that this tensor *includes* background class predictions
 290           (at class index 0).
 291         3) num_proposals: An int32 tensor of shape [batch_size] representing the
 292           number of proposals generated by the RPN. `num_proposals` allows us
 293           to keep track of which entries are to be treated as zero paddings and
 294           which are not since we always pad the number of proposals to be
 295           `self.max_num_proposals` for each image.
 296         4) proposal_boxes: A float32 tensor of shape
 297           [batch_size, self.max_num_proposals, 4] representing
 298           decoded proposal bounding boxes (in absolute coordinates).
 299         5) proposal_boxes_normalized: A float32 tensor of shape
 300           [batch_size, self.max_num_proposals, 4] representing decoded proposal
 301           bounding boxes (in normalized coordinates). Can be used to override
 302           the boxes proposed by the RPN, thus enabling one to extract box
 303           classification and prediction for externally selected areas of the
 304           image.
 305         6) box_classifier_features: a 4-D float32 tensor, of shape
 306           [batch_size, feature_map_height, feature_map_width, depth],
 307           representing the box classifier features.
 308     """
 309     image_shape_2d = tf.tile(tf.expand_dims(image_shape[1:], 0),
 310                              [image_shape[0], 1])
 311     (proposal_boxes_normalized, _, _, num_proposals, _,
 312      _) = self._postprocess_rpn(rpn_box_encodings,
 313                                 rpn_objectness_predictions_with_background,
 314                                 anchors, image_shape_2d, true_image_shapes)
 315 
 316     box_classifier_features = (
 317         self._extract_box_classifier_features(rpn_features))
 318 
 319     if self._rfcn_box_predictor.is_keras_model:
 320       box_predictions = self._rfcn_box_predictor(
 321           [box_classifier_features],
 322           proposal_boxes=proposal_boxes_normalized)
 323     else:
 324       box_predictions = self._rfcn_box_predictor.predict(
 325           [box_classifier_features],
 326           num_predictions_per_location=[1],
 327           scope=self.second_stage_box_predictor_scope,
 328           proposal_boxes=proposal_boxes_normalized)
 329     refined_box_encodings = tf.squeeze(
 330         tf.concat(box_predictions[box_predictor.BOX_ENCODINGS], axis=1), axis=1)
 331     class_predictions_with_background = tf.squeeze(
 332         tf.concat(
 333             box_predictions[box_predictor.CLASS_PREDICTIONS_WITH_BACKGROUND],
 334             axis=1),
 335         axis=1)
 336 
 337     absolute_proposal_boxes = ops.normalized_to_image_coordinates(
 338         proposal_boxes_normalized, image_shape,
 339         parallel_iterations=self._parallel_iterations)
 340 
 341     prediction_dict = {
 342         'refined_box_encodings': refined_box_encodings,
 343         'class_predictions_with_background':
 344         class_predictions_with_background,
 345         'num_proposals': num_proposals,
 346         'proposal_boxes': absolute_proposal_boxes,
 347         'box_classifier_features': box_classifier_features,
 348         'proposal_boxes_normalized': proposal_boxes_normalized,
 349         'final_anchors': absolute_proposal_boxes
 350     }
 351     if self._return_raw_detections_during_predict:
 352       prediction_dict.update(self._raw_detections_and_feature_map_inds(
 353           refined_box_encodings, absolute_proposal_boxes))
 354     return prediction_dict
 355 
 356   def regularization_losses(self):
 357     """Returns a list of regularization losses for this model.
 358 
 359     Returns a list of regularization losses for this model that the estimator
 360     needs to use during training/optimization.
 361 
 362     Returns:
 363       A list of regularization loss tensors.
 364     """
 365     reg_losses = super(RFCNMetaArch, self).regularization_losses()
 366     if self._rfcn_box_predictor.is_keras_model:
 367       reg_losses.extend(self._rfcn_box_predictor.losses)
 368     return reg_losses
 369 
 370   def updates(self):
 371     """Returns a list of update operators for this model.
 372 
 373     Returns a list of update operators for this model that must be executed at
 374     each training step. The estimator's train op needs to have a control
 375     dependency on these updates.
 376 
 377     Returns:
 378       A list of update operators.
 379     """
 380     update_ops = super(RFCNMetaArch, self).updates()
 381 
 382     if self._rfcn_box_predictor.is_keras_model:
 383       update_ops.extend(
 384           self._rfcn_box_predictor.get_updates_for(None))
 385       update_ops.extend(
 386           self._rfcn_box_predictor.get_updates_for(
 387               self._rfcn_box_predictor.inputs))
 388     return update_ops
