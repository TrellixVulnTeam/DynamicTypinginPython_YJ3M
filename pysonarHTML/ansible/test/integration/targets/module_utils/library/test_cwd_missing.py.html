  14     # 1) Execute a module which has high startup overhead prior to instantiating AnsibleModule (0.5s is enough in many cases).

  16     # 3) Remove the directory containing the playbook immediately after playbook execution ends (playbook in a temp dir).

  18     # To ease testing of this race condition the deletion of cwd is handled in this module.
  19     # This avoids race conditions in the test, including timing cwd deletion between AnsiballZ wrapper execution and AnsibleModule instantiation.
  20     # The timing issue with AnsiballZ is due to cwd checking in the wrapper when code coverage is enabled.
