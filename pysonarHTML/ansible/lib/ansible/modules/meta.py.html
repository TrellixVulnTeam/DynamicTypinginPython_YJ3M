  11 DOCUMENTATION = r'''
  12 module: meta
  13 short_description: Execute Ansible 'actions'
  14 version_added: '1.2'
  15 description:
  16     - Meta tasks are a special kind of task which can influence Ansible internal execution or state.
  17     - Meta tasks can be used anywhere within your playbook.
  18     - This module is also supported for Windows targets.
  19 options:
  20   free_form:
  21     description:
  22         - This module takes a free form command, as a string. There is not an actual option named "free form".  See the examples!
  23         - C(flush_handlers) makes Ansible run any handler tasks which have thus far been notified. Ansible inserts these tasks internally at certain
  24           points to implicitly trigger handler runs (after pre/post tasks, the final role execution, and the main tasks section of your plays).
  25         - C(refresh_inventory) (added in Ansible 2.0) forces the reload of the inventory, which in the case of dynamic inventory scripts means they will be
  26           re-executed. If the dynamic inventory script is using a cache, Ansible cannot know this and has no way of refreshing it (you can disable the cache
  27           or, if available for your specific inventory datasource (e.g. aws), you can use the an inventory plugin instead of an inventory script).
  28           This is mainly useful when additional hosts are created and users wish to use them instead of using the M(add_host) module.
  29         - C(noop) (added in Ansible 2.0) This literally does 'nothing'. It is mainly used internally and not recommended for general use.
  30         - C(clear_facts) (added in Ansible 2.1) causes the gathered facts for the hosts specified in the play's list of hosts to be cleared,
  31           including the fact cache.
  32         - C(clear_host_errors) (added in Ansible 2.1) clears the failed state (if any) from hosts specified in the play's list of hosts.
  33         - C(end_play) (added in Ansible 2.2) causes the play to end without failing the host(s). Note that this affects all hosts.
  34         - C(reset_connection) (added in Ansible 2.3) interrupts a persistent connection (i.e. ssh + control persist)
  35         - C(end_host) (added in Ansible 2.8) is a per-host variation of C(end_play). Causes the play to end for the current host without failing it.
  36     choices: [ clear_facts, clear_host_errors, end_host, end_play, flush_handlers, noop, refresh_inventory, reset_connection ]
  37     required: true
  38 notes:
  39     - C(meta) is not really a module nor action_plugin as such it cannot be overwritten.
  40     - C(clear_facts) will remove the persistent facts from M(set_fact) using C(cacheable=True),
  41       but not the current host variable it creates for the current run.
  42     - Looping on meta tasks is not supported.
  43     - Skipping C(meta) tasks using tags is not supported.
  44     - This module is also supported for Windows targets.
  45 seealso:
  46 - module: assert
  47 - module: fail
  48 author:
  49     - Ansible Core Team
  50 '''
