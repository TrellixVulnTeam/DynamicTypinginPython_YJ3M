  10 ---
  11 module: pause
  12 short_description: Pause playbook execution
  13 description:
  14   - Pauses playbook execution for a set amount of time, or until a prompt is acknowledged.
  15     All parameters are optional. The default behavior is to pause with a prompt.
  16   - To pause/wait/sleep per host, use the M(wait_for) module.
  17   - You can use C(ctrl+c) if you wish to advance a pause earlier than it is set to expire or if you need to abort a playbook run entirely.
  18     To continue early press C(ctrl+c) and then C(c). To abort a playbook press C(ctrl+c) and then C(a).
  19   - The pause module integrates into async/parallelized playbooks without any special considerations (see Rolling Updates).
  20     When using pauses with the C(serial) playbook parameter (as in rolling updates) you are only prompted once for the current group of hosts.
  21   - This module is also supported for Windows targets.
  22 version_added: "0.8"
  23 options:
  24   minutes:
  25     description:
  26       - A positive number of minutes to pause for.
  27   seconds:
  28     description:
  29       - A positive number of seconds to pause for.
  30   prompt:
  31     description:
  32       - Optional text to use for the prompt message.
  33   echo:
  34     description:
  35       - Controls whether or not keyboard input is shown when typing.
  36       - Has no effect if 'seconds' or 'minutes' is set.
  37     type: bool
  38     default: 'yes'
  39     version_added: 2.5
  40 author: "Tim Bielawa (@tbielawa)"
  41 notes:
  42       - Starting in 2.2,  if you specify 0 or negative for minutes or seconds, it will wait for 1 second, previously it would wait indefinitely.
  43       - This module is also supported for Windows targets.
  44       - User input is not captured or echoed, regardless of echo setting, when minutes or seconds is specified.
