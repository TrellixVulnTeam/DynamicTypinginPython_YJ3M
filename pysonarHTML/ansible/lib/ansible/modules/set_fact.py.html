  12 ---
  13 module: set_fact
  14 short_description: Set host facts from a task
  15 version_added: "1.2"
  16 description:
  17     - This module allows setting new variables.
  18     - Variables are set on a host-by-host basis just like facts discovered by the setup module.
  19     - These variables will be available to subsequent plays during an ansible-playbook run.
  20     - Set C(cacheable) to C(yes) to save variables across executions
  21       using a fact cache. Variables created with set_fact have different precedence depending on whether they are or are not cached.
  22     - Per the standard Ansible variable precedence rules, many other types of variables have a higher priority, so this value may be overridden.
  23     - This module is also supported for Windows targets.
  24 options:
  25   key_value:
  26     description:
  27       - The C(set_fact) module takes key=value pairs as variables to set
  28         in the playbook scope. Or alternatively, accepts complex arguments
  29         using the C(args:) statement.
  30     required: true
  31   cacheable:
  32     description:
  33       - This boolean converts the variable into an actual 'fact' which will also be added to the fact cache, if fact caching is enabled.
  34       - Normally this module creates 'host level variables' and has much higher precedence, this option changes the nature and precedence
  35         (by 7 steps) of the variable created.
  36         U(https://docs.ansible.com/ansible/latest/user_guide/playbooks_variables.html#variable-precedence-where-should-i-put-a-variable)
  37       - "This actually creates 2 copies of the variable, a normal 'set_fact' host variable with high precedence and
  38         a lower 'ansible_fact' one that is available for persistance via the facts cache plugin.
  39         This creates a possibly confusing interaction with C(meta: clear_facts) as it will remove the 'ansible_fact' but not the host variable."
  40     type: bool
  41     default: no
  42     version_added: "2.4"
  43 notes:
  44     - "The C(var=value) notation can only create strings or booleans.
  45       If you want to create lists/arrays or dictionary/hashes use C(var: [val1, val2])."
  46     - Since 'cacheable' is now a module param, 'cacheable' is no longer a valid fact name as of Ansible 2.4.
  47     - This module is also supported for Windows targets.
  48 seealso:
  49 - module: include_vars
  50 - ref: ansible_variable_precedence
  51   description: More information related to variable precedence and which type of variable wins over others.
  52 author:
  53 - Dag Wieers (@dagwieers)
  54 '''
