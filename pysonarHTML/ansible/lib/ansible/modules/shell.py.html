  15 DOCUMENTATION = r'''
  16 ---
  17 module: shell
  18 short_description: Execute shell commands on targets
  19 description:
  20      - The C(shell) module takes the command name followed by a list of space-delimited arguments.
  21      - Either a free form command or C(cmd) parameter is required, see the examples.
  22      - It is almost exactly like the M(command) module but runs
  23        the command through a shell (C(/bin/sh)) on the remote node.
  24      - For Windows targets, use the M(win_shell) module instead.
  25 version_added: "0.2"
  26 options:
  27   free_form:
  28     description:
  29       - The shell module takes a free form command to run, as a string.
  30       - There is no actual parameter named 'free form'.
  31       - See the examples on how to use this module.
  32     type: str
  33   cmd:
  34     type: str
  35     description:
  36       - The command to run followed by optional arguments.
  37   creates:
  38     description:
  39       - A filename, when it already exists, this step will B(not) be run.
  40     type: path
  41   removes:
  42     description:
  43       - A filename, when it does not exist, this step will B(not) be run.
  44     type: path
  45     version_added: "0.8"
  46   chdir:
  47     description:
  48       - Change into this directory before running the command.
  49     type: path
  50     version_added: "0.6"
  51   executable:
  52     description:
  53       - Change the shell used to execute the command.
  54       - This expects an absolute path to the executable.
  55     type: path
  56     version_added: "0.9"
  57   warn:
  58     description:
  59       - Whether to enable task warnings.
  60     type: bool
  61     default: yes
  62     version_added: "1.8"
  63   stdin:
  64     description:
  65       - Set the stdin of the command directly to the specified value.
  66     type: str
  67     version_added: "2.4"
  68   stdin_add_newline:
  69     description:
  70       - Whether to append a newline to stdin data.
  71     type: bool
  72     default: yes
  73     version_added: "2.8"
  74 notes:
  75   - If you want to execute a command securely and predictably, it may be
  76     better to use the M(command) module instead. Best practices when writing
  77     playbooks will follow the trend of using M(command) unless the C(shell)
  78     module is explicitly required. When running ad-hoc commands, use your best
  79     judgement.
  80   - Check mode is supported when passing C(creates) or C(removes). If running
  81     in check mode and either of these are specified, the module will check for
  82     the existence of the file and report the correct changed status. If these

  84   - To sanitize any variables passed to the shell module, you should use
  85     C({{ var | quote }}) instead of just C({{ var }}) to make sure they
  86     do not include evil things like semicolons.
  87   - An alternative to using inline shell scripts with this module is to use

  89   - For rebooting systems, use the M(reboot) or M(win_reboot) module.

 100 EXAMPLES = r'''
 101 - name: Execute the command in remote shell; stdout goes to the specified file on the remote
 102   shell: somescript.sh >> somelog.txt
 103 
 104 - name: Change the working directory to somedir/ before executing the command
 105   shell: somescript.sh >> somelog.txt
 106   args:
 107     chdir: somedir/
 108 
 109 # You can also use the 'args' form to provide the options.
 110 - name: This command will change the working directory to somedir/ and will only run when somedir/somelog.txt doesn't exist
 111   shell: somescript.sh >> somelog.txt
 112   args:
 113     chdir: somedir/
 114     creates: somelog.txt
 115 
 116 # You can also use the 'cmd' parameter instead of free form format.
 117 - name: This command will change the working directory to somedir/
 118   shell:
 119     cmd: ls -l | grep log
 120     chdir: somedir/
 121 
 122 - name: Run a command that uses non-posix shell-isms (in this example /bin/sh doesn't handle redirection and wildcards together but bash does)
 123   shell: cat < /tmp/*txt
 124   args:
 125     executable: /bin/bash
 126 
 127 - name: Run a command using a templated variable (always use quote filter to avoid injection)
 128   shell: cat {{ myfile|quote }}
 129 
 130 # You can use shell to run other executables to perform actions inline
 131 - name: Run expect to wait for a successful PXE boot via out-of-band CIMC
 132   shell: |
 133     set timeout 300
 134     spawn ssh admin@{{ cimc_host }}
 135 
 136     expect "password:"
 137     send "{{ cimc_password }}\n"
 138 
 139     expect "\n{{ cimc_name }}"
 140     send "connect host\n"
 141 
 142     expect "pxeboot.n12"
 143     send "\n"
 144 
 145     exit 0
 146   args:
 147     executable: /usr/bin/expect
 148   delegate_to: localhost
 149 
 150 # Disabling warnings
 151 - name: Using curl to connect to a host via SOCKS proxy (unsupported in uri). Ordinarily this would throw a warning
 152   shell: curl --socks5 localhost:9000 http://www.ansible.com
 153   args:
 154     warn: no
 155 '''
