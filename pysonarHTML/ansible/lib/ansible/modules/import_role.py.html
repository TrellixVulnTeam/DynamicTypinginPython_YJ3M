  10 DOCUMENTATION = r'''
  11 ---
  12 author: Ansible Core Team (@ansible)
  13 module: import_role
  14 short_description: Import a role into a play
  15 description:
  16   - Much like the C(roles:) keyword, this task loads a role, but it allows you to control when the role tasks run in
  17     between other tasks of the play.
  18   - Most keywords, loops and conditionals will only be applied to the imported tasks, not to this statement itself. If
  19     you want the opposite behavior, use M(include_role) instead.
  20 version_added: '2.4'
  21 options:
  22   name:
  23     description:
  24       - The name of the role to be executed.
  25     type: str
  26     required: true
  27   tasks_from:
  28     description:
  29       - File to load from a role's C(tasks/) directory.
  30     type: str
  31     default: main
  32   vars_from:
  33     description:
  34       - File to load from a role's C(vars/) directory.
  35     type: str
  36     default: main
  37   defaults_from:
  38     description:
  39       - File to load from a role's C(defaults/) directory.
  40     type: str
  41     default: main
  42   allow_duplicates:
  43     description:
  44       - Overrides the role's metadata setting to allow using a role more than once with the same parameters.
  45     type: bool
  46     default: yes
  47   handlers_from:
  48     description:
  49       - File to load from a role's C(handlers/) directory.
  50     type: str
  51     default: main
  52     version_added: '2.8'
  53 notes:
  54   - Handlers are made available to the whole play.
  55   - Since Ansible 2.7 variables defined in C(vars) and C(defaults) for the role are exposed at playbook parsing time.
  56     Due to this, these variables will be accessible to roles and tasks executed before the location of the
  57     M(import_role) task.
  58   - Unlike M(include_role) variable exposure is not configurable, and will always be exposed.
