   8 DOCUMENTATION = r'''
   9 ---
  10 module: script
  11 version_added: "0.9"
  12 short_description: Runs a local script on a remote node after transferring it
  13 description:
  14      - The C(script) module takes the script name followed by a list of space-delimited arguments.
  15      - Either a free form command or C(cmd) parameter is required, see the examples.
  16      - The local script at path will be transferred to the remote node and then executed.
  17      - The given script will be processed through the shell environment on the remote node.
  18      - This module does not require python on the remote system, much like the M(raw) module.
  19      - This module is also supported for Windows targets.
  20 options:
  21   free_form:
  22     description:
  23       - Path to the local script file followed by optional arguments.
  24   cmd:
  25     type: str
  26     description:
  27       - Path to the local script to run followed by optional arguments.
  28   creates:
  29     description:
  30       - A filename on the remote node, when it already exists, this step will B(not) be run.
  31     version_added: "1.5"
  32   removes:
  33     description:
  34       - A filename on the remote node, when it does not exist, this step will B(not) be run.
  35     version_added: "1.5"
  36   chdir:
  37     description:
  38       - Change into this directory on the remote node before running the script.
  39     version_added: "2.4"
  40   executable:
  41     description:
  42       - Name or path of a executable to invoke the script with.
  43     version_added: "2.6"
  44 notes:
  45   - It is usually preferable to write Ansible modules rather than pushing scripts. Convert your script to an Ansible module for bonus points!
  46   - The C(ssh) connection plugin will force pseudo-tty allocation via C(-tt) when scripts are executed. Pseudo-ttys do not have a stderr channel and all
  47     stderr is sent to stdout. If you depend on separated stdout and stderr result keys, please switch to a copy+command set of tasks instead of using script.
  48   - If the path to the local script contains spaces, it needs to be quoted.
  49   - This module is also supported for Windows targets.
  50 seealso:
  51 - module: shell
  52 - module: win_shell
  53 author:
  54     - Ansible Core Team
  55     - Michael DeHaan
  56 extends_documentation_fragment:
  57     - decrypt
  58 '''
