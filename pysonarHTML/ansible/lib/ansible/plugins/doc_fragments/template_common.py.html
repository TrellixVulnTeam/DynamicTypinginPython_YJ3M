  14 description:
  15 - Templates are processed by the L(Jinja2 templating language,http://jinja.pocoo.org/docs/).
  16 - Documentation on the template formatting can be found in the
  17   L(Template Designer Documentation,http://jinja.pocoo.org/docs/templates/).
  18 - Additional variables listed below can be used in templates.
  19 - C(ansible_managed) (configurable via the C(defaults) section of C(ansible.cfg)) contains a string which can be used to
  20   describe the template name, host, modification time of the template file and the owner uid.
  21 - C(template_host) contains the node name of the template's machine.
  22 - C(template_uid) is the numeric user id of the owner.
  23 - C(template_path) is the path of the template.
  24 - C(template_fullpath) is the absolute path of the template.
  25 - C(template_destpath) is the path of the template on the remote system (added in 2.8).
  26 - C(template_run_date) is the date that the template was rendered.
  27 options:
  28   src:
  29     description:
  30     - Path of a Jinja2 formatted template on the Ansible controller.
  31     - This can be a relative or an absolute path.
  32     - The file must be encoded with C(utf-8) but I(output_encoding) can be used to control the encoding of the output
  33       template.
  34     type: path
  35     required: yes
  36   dest:
  37     description:
  38     - Location to render the template to on the remote machine.
  39     type: path
  40     required: yes
  41   newline_sequence:
  42     description:
  43     - Specify the newline sequence to use for templating files.
  44     type: str
  45     choices: [ '\n', '\r', '\r\n' ]
  46     default: '\n'
  47     version_added: '2.4'
  48   block_start_string:
  49     description:
  50     - The string marking the beginning of a block.
  51     type: str
  52     default: '{%'
  53     version_added: '2.4'
  54   block_end_string:
  55     description:
  56     - The string marking the end of a block.
  57     type: str
  58     default: '%}'
  59     version_added: '2.4'
  60   variable_start_string:
  61     description:
  62     - The string marking the beginning of a print statement.
  63     type: str
  64     default: '{{'
  65     version_added: '2.4'
  66   variable_end_string:
  67     description:
  68     - The string marking the end of a print statement.
  69     type: str
  70     default: '}}'
  71     version_added: '2.4'
  72   trim_blocks:
  73     description:
  74     - Determine when newlines should be removed from blocks.
  75     - When set to C(yes) the first newline after a block is removed (block, not variable tag!).
  76     type: bool
  77     default: yes
  78     version_added: '2.4'
  79   lstrip_blocks:
  80     description:
  81     - Determine when leading spaces and tabs should be stripped.
  82     - When set to C(yes) leading spaces and tabs are stripped from the start of a line to a block.
  83     - This functionality requires Jinja 2.7 or newer.
  84     type: bool
  85     default: no
  86     version_added: '2.6'
  87   force:
  88     description:
  89     - Determine when the file is being transferred if the destination already exists.
  90     - When set to C(yes), replace the remote file when contents are different than the source.
  91     - When set to C(no), the file will only be transferred if the destination does not exist.
  92     type: bool
  93     default: yes
  94   output_encoding:
  95     description:
  96     - Overrides the encoding used to write the template file defined by C(dest).
  97     - It defaults to C(utf-8), but any encoding supported by python can be used.
  98     - The source template file must always be encoded using C(utf-8), for homogeneity.
  99     type: str
 100     default: utf-8
 101     version_added: '2.7'
 102 notes:
 103 - Including a string that uses a date in the template will result in the template being marked 'changed' each time.
 104 - Since Ansible 0.9, templates are loaded with C(trim_blocks=True).
 105 - >
 106   Also, you can override jinja2 settings by adding a special header to template file.
 107   i.e. C(#jinja2:variable_start_string:'[%', variable_end_string:'%]', trim_blocks: False)
 108   which changes the variable interpolation markers to C([% var %]) instead of C({{ var }}).
 109   This is the best way to prevent evaluation of things that look like, but should not be Jinja2.
 110 - Using raw/endraw in Jinja2 will not work as you expect because templates in Ansible are recursively
 111   evaluated.
 112 - To find Byte Order Marks in files, use C(Format-Hex <file> -Count 16) on Windows, and use C(od -a -t x1 -N 16 <file>)
 113   on Linux.
