    ref: Stop double-processing minidumps in API (#13222)
    
    Adds an option to disable minidump processing in the API. Minidumps will still be fully processed by symbolicator during event processing. Due to this change, invalid minidumps will show up in the issue stream.
    
    The option is random-sampled and can be forced for specific projects.
