    fix(ui) Rename none to default (#18645)
    
    The name 'none' is misleading as you don't get no graph, you get
    a default 'all things' view. Saved queries and bookmarks could contain
    'none' but because that is now an invalid option the fallback is
    'default' so there should be no user impact of this change.
