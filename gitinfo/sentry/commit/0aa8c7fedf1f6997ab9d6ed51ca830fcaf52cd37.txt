    Dubble down on simplejson inside the riak client
    
    This fixes an issue with `json` not being able to `load` a value that it
    has previously `dump`'d. But `simplejson` can! This also only affects
    older 2.7.x series.
    
    ```
    >>> json.loads(json.dumps({u'': u'\ud800'}))
    ```
    
    Does not work on 2.7.3, but _does_ work on 2.7.10. Unfortunately, we run
    2.7.3 in production right now, as do probably lots of others.
    
    See also: https://bugs.python.org/issue11489
    
    Fixes GH-2121
