    ref(onboarding-wizard): Improve interface to wizard (#17073)
    
    - The types for OnboardingTasks have been split into
       OnboardingTaskDescriptor and OnboardingTaskStatus. OnboardingTask is
       now an intersection of the two types
    
     - `featureLocation` has been renamed to `actionType` and now accepts
       the values [`action`, `app`, and `external`].
    
       `app` replaces project and organization and now uses the
       navigation.nanavigateTo action to determine the needed slugs using
       the ContextPickerModal.
    
       `external` replaces `absolute` and is simply a more consistent name
       (as this now also uses `ExternalLink`)
    
       `action` replaces `modal` and allows a `action` option to be
       specified on the task, `location` is now always type `string`.
    
     - The ProjectChooser view has been completely removed since this is now
       replaced by the context picker modal.
    
     - TodoItem and TodoList have both been converted to typescript
    
     - TodoItem has lost it's ability to 'expand' items, as they are now
       always visually expanded.
    
     - TodoItem has had some visual stylistic improvements made
    
       * The entire item is now clickable
       * The text is now variants of gray
       * When a task is complete, the checkmark circle is now also green
