    auth: add nonce to user sessions (#4466)
    
    This nonce is used to invalidate other logged in sessions. If an action
    is taken that alters an account's security settings, we take action to
    rotate the nonce and forcibly logout all other existing sessions. Right
    now, this means when a user changes their password or adds a 2fa method,
    we bump the session nonce causing other sessions to need to be
    reauthenticated. This also opens the door for exposing a way to just
    forcibly logout all sessions if you suspect suspicious activity, but
    this isn't exposed through any public API.
