    refs: Stop using `TransactionTestCase` in some cases to speed up test running time
    
    Tests in tests/sentry/event_manager/test_event_manager.py take about 2 minutes to run with
    `TransactionTestCase`, and about 15-20 seconds with just `TestCase`. I'm not sure that we're really
    testing any specific transactional behaviour here, so it seems like it might be ok to switch over.
