    ref(onboarding): Remove unneeded API usages in onboarding setup (#12902)
    
    This change introduces various fixes and improvements to the project
    setup instructions. The changes are outlined below:
    
     - The ProjectDocsEndpoint is completely removed. The frontend already
       has access to all information provided by this endpoint via the
       `app/data/platforms` module, which pulls data from the integration
       documentation files themselves (just as the endpoint did).
    
       The keys are also available through the usual project/keys endpoint.
    
     - The ProjectDocsContext component has been completely removed. This
       component originally provided the data from the ProjectDocsEndpoint,
       however since this data is already available to the frontend, we can
       just import the platform data where needed.
    
       This module also introduced complexity in understanding where
       properties were being passed from, as it made use of
       React.cloneElement.
    
       - OnboardingConfigure no longer uses ProjectDocsContext and instead
         directly renders it's children. It's children will pull in
         platforms when needed.
    
       - The project installation routes are no longer wrapped with
         ProjectDocsContext.
    
       - The following components no longer require the ProjectDocsContext:
    
         - GettingStarted (Part of projectInstall)
         - ProjectInstallPlatform
         - ProjectInstallOverview
    
     - ProjectInstallOverview has been refactored to no longer need the
       platformData prop passed from ProjectDocsContext. This component will
       now receive platform information from the data/platform module.
    
       The DSN keys are loaded from the project/keys endpoint
    
       This component has also had various stylistic improvements:
    
       - The project public/private dsn keys are now rendered using styled
         components and no longer look like disabled inputs.
    
       - Various other old-style css classes have been replaced with
         components such as Buttons and Links.
    
     - The ProjectInstallPlatform component (the *actual* docs rendering
       component) has been heavily refactored.
    
       - The component is now class based.
    
       - The platform documentation loading has been extracted to the
         actionCreators/projet module. And now uses async style calls.
    
       - linkPath has been completely removed along with the getPlatformLink
         method as these were completely unused.
