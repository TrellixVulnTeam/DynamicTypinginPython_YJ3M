    fix(test) Fixate time better in discover tests (#17247)
    
    Don't use relative time for test events. Fixate the time parts so
    breadcrumbs have stable output.
    
    Use dynamicValue on breadcrumbs. By fixating the datetime to address
    breadcrumbs, the issue first/last seen times would shift. This approach
    will ensure that we don't have any more breadcrumb value shifts in the
    future.
