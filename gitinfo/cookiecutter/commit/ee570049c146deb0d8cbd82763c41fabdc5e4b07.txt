    Remove cookiecutter import from setup.py. Use dev versioning
    
    Keeping the in-development version number the same as the PyPI version
    number results in incorrect dependencies being installed on Windows.
    This commit bumps the version to 0.7.0.dev0.
    
    Also, importing a package in its setup.py causes circular imports in
    some situations, so let's not do that.
