    Man page section about `tox.ini` updated (no need for `pipenv run`)
    
    As `pipenv` runs in `command` section in `tox` activated virtualenv,
    there is no need to run `pipenv run <some_command>` and commands can
    be run directly as `<some_command>`. This also relaxes the requirement
    for passing `HOME` environment variable into `pipenv run` call.
