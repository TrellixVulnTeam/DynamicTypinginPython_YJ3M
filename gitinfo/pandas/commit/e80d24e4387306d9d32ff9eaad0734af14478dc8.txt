    BLD: add a 2.7 vbench build to the travis build matrix
    
    a seperate job is spawned, which is allowed to fail without
    failing the entire build.
    the results are posted to a public gist as a json file.
    
    helper functions in vb_suite/perf_HEAD.py accept a travis
    build number, locate the VBENCH job, fetch the log through
    the Travis API, extract the gist raw_url, fetch the results
    and decode the json into a dict and then a pandas dataframe.
    FTW.
    
    It remains to be seen whether the results are consistent
    enough between builds over time to make this useful.
